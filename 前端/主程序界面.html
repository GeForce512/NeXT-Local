NeXT 使用教程
⚡
快速开始
环境检测
进入「设置」页面，点击「开始检测」。程序会自动识别你的显卡并安装 PyTorch 等运行库。只需首次使用时操作一次。
下载模型
进入「模型」页面，选择模型版本（0.8B 轻量 / 4B 基础），点击「开始下载」。模型保存在程序目录下的 models 文件夹。
开始对话
进入「对话」页面，等待顶部胶囊显示为绿色（模型已加载），即可输入消息开始聊天。
模型下载、LoRA 权重、训练数据集全部存放在程序同级目录下，不会占用 C 盘空间。
⚠
️ 重要注意事项
请避免频繁并发操作！NeXT 是单进程应用，不支持同时进行多个耗时操作（如同时训练和切换 LoRA）。请在当前操作完成后再进行下一步，否则可能导致程序卡死或数据损坏。
训练过程中不要关闭程序，否则可能导致 LoRA 权重损坏。
点击「关闭模型」后，请等待侧边栏按钮从"关闭中..."恢复为"关闭模型"再进行其他操作。
切换 LoRA 时，如果提示"请等待模型关闭完成"，说明模型正在关闭中，请稍后再试。
增量训练需要选择一个已有的 LoRA 作为基础，适合在之前训练结果上继续优化。
训练完成后，在「模型」页面的 LoRA 权重列表中选中训练好的权重，点击「切换选中」即可在对话中使用。
如果对话时模型回答质量很差，可能是 LoRA 没有正确加载。检查顶部胶囊是否显示为绿色并带有 LoRA 名称。

硬件要求
显卡显存
最低需要 4GB 显存。推荐 6GB 以上以获得更好的训练体验。
矿卡 / 计算卡支持
P106、30HX、40HX 等计算卡可以使用，前提是已正确安装 NVIDIA 官方驱动，能被 CUDA 正常识别。
安装驱动后可在命令行运行 nvidia-smi 确认显卡是否被识别
内存
建议 8GB 以上。训练时模型和数据处理会占用较多内存。
⚙
训练参数详解
学习率 (Learning Rate)
控制模型每次学习时"步子迈多大"。太大容易学歪（loss 震荡），太小则学得太慢。默认 0.0003 是一个比较安全的值。
一般不需要改。如果训练 loss 不下降，可以尝试调小到 0.0001。
训练轮数 (Epochs)
数据集要被完整学几遍。1 轮 = 把所有数据过一遍。轮数越多学得越深，但也越容易"死记硬背"（过拟合）。
小数据集（几百条）建议 3-5 轮；大数据集（几千条以上）2-3 轮就够了。
最大序列长度 (Max Sequence Length)
模型一次能"看到"的最长文本长度，单位是 token（大致等于汉字数）。设得越大，模型能学习越长的对话，但显存占用也越高。
短对话（问答对）设 256-512；长文本（文章、故事）设 1024-2048。显存不够就调小这个值。
批次大小 (Batch Size)
每次同时处理多少条数据。越大训练越快越稳定，但显存占用成倍增长。
4GB 显存建议设为 1；6GB 设为 2；8GB 以上可以设为 4。这是影响显存最大的参数。
优化器 (Optimizer)
决定模型如何根据误差来调整自己。三种可选：
AdamW（默认）— 最通用，适合绝大多数场景，收敛稳定。
SGD — 传统方法，有时在特定任务上效果更好，但需要更细心调学习率。
Adafactor — 省显存，适合显存紧张的情况，但训练速度稍慢。
不知道选哪个就用 AdamW，它在 95% 的情况下都是最好的选择。
正则化系数 (Weight Decay)
防止模型"死记硬背"训练数据的惩罚力度。值越大，模型越倾向于学通用规律而不是逐字背诵。
默认 0.01 即可。如果训练后模型只会重复训练数据的内容（过拟合），可以调大到 0.05。
LoRA Rank (r)
LoRA 适配器的"容量"。r 越大，模型能学到的东西越多，但训练越慢、文件越大。
简单任务（风格模仿）r=8 就够；复杂任务（专业知识）r=16-32；r=64 以上一般没必要。
LoRA Alpha
LoRA 的缩放系数，控制适配器对模型的影响强度。通常设为 r 的 2 倍（Alpha = 2 × Rank）。
保持 Alpha = 2 × Rank 的比例即可。比如 r=16 时 Alpha=32，r=8 时 Alpha=16。

推荐训练方案
以下是三套预设方案，根据你的需求选择一套，在「设置」页面按对应参数填写即可。
快速
快速验证方案
适合快速测试数据集质量、验证训练流程是否正常。训练速度快，但效果一般。
学习率：0.0005
训练轮数：2
序列长度：256
批次大小：1
优化器：AdamW
Weight Decay：0.01
LoRA Rank：8
LoRA Alpha：16
常规
常规训练方案
平衡速度与效果的默认方案，适合大多数场景。日常微调推荐用这套。
学习率：0.0003
训练轮数：4
序列长度：512
批次大小：2
优化器：AdamW
Weight Decay：0.01
LoRA Rank：16
LoRA Alpha：32
质量
高质量训练方案
追求最佳训练效果，适合正式产出。训练时间较长，需要 6GB 以上显存。
学习率：0.0002
训练轮数：5
序列长度：1024
批次大小：4
优化器：AdamW
Weight Decay：0.01
LoRA Rank：32
LoRA Alpha：64
显存不足时优先调小「批次大小」和「序列长度」，这两个对显存影响最大。

数据集格式
JSONL 格式（推荐）
每行一条 JSON 数据，包含 messages 数组。示例：
{"messages":[{"role":"user","content":"你好"},{"role":"assistant","content":"你好！有什么可以帮你的？"}]}
TXT 格式
每两行为一组对话：第一行是用户问题，第二行是模型回答。空行分隔不同对话。
CSV 格式
两列：第一列是用户问题，第二列是模型回答。首行为标题行（会被跳过）。
训练时会自动过滤空答案和重复数据，无需手动清洗。
