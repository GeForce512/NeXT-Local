# NeXT-Local

一个本地大模型推理和微调的桌面端工作台。
没有黑乎乎的命令行，没有动辄几个 GB 的臃肿整合包。主程序只有几十 MB，双击就能跑。

## 为什么写这个项目

现在想在本机跑个大模型，要么面对 Ollama 复杂的命令行配置，要么下载解压慢得要死的“懒人包”。
我想做一个干净、轻量的客户端，把环境配置、模型下载、LoRA 微调和推理全塞进一个 GUI 里，让不懂代码的人也能直接用。

## 核心实现与踩坑记录

* **主程序与推理进程物理隔离**
  主程序（`主程序入口.py`）只负责 PyQt5 界面和调度；推理后端（`推理.py`）跑在独立的 Flask 进程里。两者通过本地 API 通信，UI 永远不会因为模型加载或 OOM 而卡死。
* **PyInstaller 打包 WebEngine 的“巨坑”**
  用 PyInstaller 打包 PyQtWebEngine 经常会缺 `.pak` 或 `swiftshader` 导致界面白屏。我在 `build.py` 里写了一个递归搜索脚本，会自动去系统的 `Adobe` 和 `Norton` 等目录里把这些缺失的底层 CEF 文件扒出来补齐。
* **带重试的弱网下载器**
  基于 ModelScope 的 `snapshot_download` 封装了下载器。因为学校机房网络经常波动，我加了 3 次指数级退避重试逻辑，断网了也能接着下，模型统一丢进 `models/` 目录。
* **把 Temperature 映射到 UI 颜色**
  写了个前端 JS 插值算法，把大模型的 `Temperature` (发散度) 参数和界面背景色绑定了。问代码时界面是冷蓝色，写小说时是暖橙色。没什么实际用途，但看着很爽。

## 怎么跑起来

### 开发环境运行
```bash
# 1. 安装基础依赖
pip install -r requirements.txt

# 2. 构建本地 AI 环境 (自动处理 PyTorch 等)
python 环境.py

# 3. 下载基座模型 (支持断点续传)
python 模型下载.py

# 4. 启动服务 (需要开两个终端)
python 推理.py        # 终端 1：启动后端
python 主程序入口.py   # 终端 2：启动 UI
